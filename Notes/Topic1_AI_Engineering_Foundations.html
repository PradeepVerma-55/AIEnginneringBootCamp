What is AI Engineering?

🔑 Real-world analogy

Think of traditional software engineering as building a car from scratch.
An AI Engineer gets a pre-built, highly intelligent engine (the LLM) and their job is to: wire it up to real systems, make it useful for real users, and ship it reliably at scale.

You don't train the model. You orchestrate it.

Core Pattern — It's Just an API Call

C# — Calling a Payment API (what you know)
// Familiar HTTP call pattern
var payload = new StringContent(
    JsonSerializer.Serialize(new { amount = 100 }),
    Encoding.UTF8, "application/json"
);
var response = await httpClient.PostAsync(
    "https://payment-api.com/charge", payload
);
var result = await response.Content.ReadAsStringAsync();
Console.WriteLine(result);
Python — Calling an LLM API (same concept!)
from openai import OpenAI

client = OpenAI()

response = client.chat.completions.create(
    model="gpt-4",
    messages=[
        {"role": "user", "content": "Summarise in 3 bullet points"}
    ]
)
print(response.choices[0].message.content)
🔑 Same mental model: call → response. Only difference — the response is intelligent text, not structured data.

python -m venv .venv

# Windows
.venv\Scripts\activate

# Mac / Linux
source .venv/bin/activate

pip install openai python-dotenv fastapi uvicorn
Python — Load API key safely
from dotenv import load_dotenv
import os

load_dotenv()
api_key = os.getenv("GROQ_API_KEY")
C# — Same idea (you already know this!)
var apiKey = configuration["GROQ_API_KEY"];
// or
var apiKey = Environment.GetEnvironmentVariable("GROQ_API_KEY");
🔵 C# parallel: venv = your .csproj NuGet boundary. Each project gets isolated dependencies. Same concept, different syntax.

Key Concepts Glossary

LLM
Large Language Model — the AI brain (GPT-4, Claude, Gemini)
API
A way to call a service over HTTP — you already know this!
Prompt
The instruction or question you send to the LLM
Token
~0.75 words — how LLMs measure and charge for text
Context Window
Max tokens an LLM can "see" at once in one call
Inference
Running the model to generate an answer
RAG
Retrieval-Augmented Generation — LLM + your private data
Agent
An LLM that can plan, use tools, and take actions
